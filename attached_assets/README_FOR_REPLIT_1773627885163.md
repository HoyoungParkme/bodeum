# Replit Prototype Handoff

## 목적

이 폴더는 `ingan.ai`의 디자인 프로토타입 작업을 위해 정리한 핸드오프 패키지입니다.

현재 목표는 아래와 같습니다.

- 전체 서비스 구현이 아니라 `프론트엔드 프로토타입` 제작
- 현재의 기능 흐름을 유지하면서 `디자인 품질` 개선
- 이후 이 저장소에서 실제 구현 작업을 이어갈 수 있게 구조를 맞추는 것

## 제품 한 줄 설명

Big Five 기반 100문항 설문과 챗봇 심화 대화를 통해 성향 리포트를 보여주는 웹 서비스입니다.

## 현재 확정된 핵심 조건

- 분석 모델: Big Five
- 입력 흐름: `설문 -> 챗봇 -> 최종 리포트`
- 자기 분석과 동의 기반 타인 분석 지원
- 타인 분석은 상대방 본인이 직접 응답
- 서비스 톤은 `따뜻한 코치형`
- 초기 MVP는 유료화 없음
- 인스타그램 연동 없음

## Replit에 부탁하는 작업 범위

아래 범위까지만 집중해 주세요.

- 디자인 시스템 방향 제안
- 랜딩 페이지 개선
- 설문 시작 화면
- 설문 진행 화면
- 챗봇 화면
- 최종 리포트 화면
- 타인 분석 초대/동의 화면
- 모바일/데스크톱 반응형 고려

## 이번 단계에서 중요하지 않은 것

- 백엔드 로직 완성
- 실제 DB 연동
- 실제 결제
- 실제 LLM 연결
- 실제 인증

즉, 이번 단계에서는 `화면 설계와 프로토타입 완성도`가 가장 중요합니다.

## 디자인 방향

지금 있는 화면은 구조 확인용 뼈대이며, 디자인 완성도가 낮습니다.

원하는 방향:

- 너무 차갑거나 의료 앱처럼 보이지 않을 것
- 너무 가볍거나 테스트 사이트처럼 보이지 않을 것
- 따뜻하고 차분한 코치형 인상
- 리포트 화면은 읽기 쉽고 신뢰감 있게
- 설문 100문항이 길어 보여도 부담스럽지 않게

피하고 싶은 방향:

- 기본 템플릿 느낌
- 과한 그라데이션 남발
- 의미 없는 카드 나열
- 유치한 심리테스트 느낌

## 권장 산출물

- 화면별 프로토타입
- 공통 UI 스타일 가이드
- 색상/타이포/카드/버튼 기준
- 설문/챗봇/리포트 흐름에 맞는 컴포넌트 구조 제안
